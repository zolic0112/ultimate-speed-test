## 📝 需要做的步骤

### 第 1 步：建立你的 3D 模型 🎨

**需要时间：** 1-3 小时（取决于设计复杂度）

**步骤：**
1. 在 Blender 中创建 6 个独特的奖牌设计
2. 为每个等级应用不同的颜色和风格
3. 参考 `MEDALS_SETUP.md` 的设计指南

**设计提示：**
- S 级：极速、金色、发光（反光率 100%）
- F 级：危急、深灰色、破损（反光率 < 20%）
- 其他级别：在之间逐步过渡

### 第 2 步：导出为 glTF 2.0 📤

**需要时间：** 10 分钟

**步骤：**
1. 在 Blender 中选择每个模型
2. File → Export → glTF 2.0 (.glb)
3. 启用关键选项：
   - ✅ Format: Binary (.glb)
   - ✅ Include Materials
   - ✅ Draco compression (可选但推荐)
4. 保存为 `medal-{grade}.glb` 格式

**文件大小目标：**
- 单个模型：< 200 KB
- 全部 6 个：< 2.5 MB

### 第 3 步：集成到应用 🔧

**需要时间：** 5 分钟

**步骤：**
1. 复制 6 个 `.glb` 文件到 `app/medals/`
2. 在 `index.html` 中添加 GLTFLoader 脚本：
   ```html
   <script src="https://cdn.jsdelivr.net/npm/three@r128/examples/jsm/loaders/GLTFLoader.js"></script>
   ```
3. 验证 `medal.js` 中的路径配置（默认已正确）
4. （可选）调整模型缩放或位置

### 第 4 步：测试 🧪

**需要时间：** 10 分钟

**步骤：**
1. 打开浏览器开发工具（F12）
2. 运行完整的速度测试
3. 检查浏览器控制台输出：
   ```
   [medal] Loading model for grade S from: medals/medal-s.glb
   [medal] ✓ Successfully loaded model for grade S
   ```
4. 验证奖牌显示和交互（旋转、缩放）
5. 在多个浏览器和设备上测试

### 第 5 步：优化和部署 🚀

**需要时间：** 20 分钟

**步骤：**
1. 使用 Draco 压缩优化文件大小
2. 测试加载时间（应 < 2 秒 在 3G 网络）
3. 监控文件大小（总 < 2.5 MB）
4. 部署到生产环境
5. 监控用户反馈

## 🐛 故障排除快速参考

### 问题：模型没有显示

```
浏览器控制台检查：
1. 查找 "GLTFLoader not available" 
   → 在 index.html 中加载脚本

2. 查找 "404 Not Found"
   → 检查文件路径和文件是否存在

3. 查找其他加载错误
   → 验证 .glb 文件是否有效
```

### 问题：模型太大/太小

```
在 medal.js 的 _swapModel() 中调整：
this.body.scale.multiplyScalar(1.2);  // 放大 20%
```

### 问题：模型方向不对

```
在 medal.js 中添加旋转：
this.body.rotation.x = Math.PI / 2;   // 旋转 90°
```
